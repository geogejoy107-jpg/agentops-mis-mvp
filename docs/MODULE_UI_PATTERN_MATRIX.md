# Module UI Pattern Matrix

_Audit date: 2026-06-14_

This matrix translates the open-source UI reference audit into concrete AgentOps MIS module decisions. The references guide interaction patterns only; AgentOps MIS remains the authority system for agents, tasks, runs, tool calls, approvals, memory, evaluations, audit, runtime connectors, external bases and templates.

## Pixel Operating Map translation

The Pixel Operating Map should not duplicate the formal pages above. It should be the clickable visual layer that helps users see where work is happening and jump to the correct ledger/table. Each zone on the map links to one target route, and each agent sprite is derived from AgentOps MIS state rather than from a third-party virtual office runtime.

## Borrow / reimplement / avoid rules

- **Borrow:** task lanes, status chips, trace score cards, connector cards, audit filters, human approval gates and memory review concepts.
- **Reimplement:** all React components, CSS, zone geometry, sprite placeholders, routing, data mapping and MIS-specific labels.
- **Avoid:** copying external product code, AGPL/BSL/fair-code UI internals, third-party art, paid tilesets, branded characters and any architecture where an external runtime becomes the authority system.
